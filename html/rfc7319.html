IANA Considerations for Connectivity Fault Management (CFM) Code Points

接続障害管理（CFM）コードポイントのためのIANAの考慮事項

Abstract

抽象

IEEE 802.1 has specified Connectivity Fault Management (CFM) Operations, Administration, and Maintenance (OAM) facilities. CFM messages are structured with an OpCode field and have provision for the inclusion of TLV-structured information. IEEE 802.1 has allocated blocks of CFM OpCodes and TLV Types to the IETF. This document specifies the IANA considerations for the assignment of values from these blocks.

IEEE 802.1は、接続障害管理（CFM）運用、管理、および保守（OAM）施設を指定しています。 CFMメッセージは、オペコードフィールドで構成し、TLV構造の情報を含めるための規定を持っています。 IEEE 802.1は、IETFへのCFMのオペコードとTLVタイプのブロックを割り当てています。この文書では、これらのブロックからの値の割り当てのためにIANA問題を指定します。

1. Introduction
1. はじめに

The IEEE 802.1 Working Group has specified Connectivity Fault Management (CFM) [802.1Q] OAM [RFC6291] facilities. CFM messages are structured with an OpCode field and have provision for the inclusion of TLV-structured information.

IEEE 802.1ワーキンググループは、接続障害管理（CFM）[802.1Q] OAM [RFC6291]の施設を指定しています。 CFMメッセージは、オペコードフィールドで構成し、TLV構造の情報を含めるための規定を持っています。

IEEE 802.1 has allocated the block of 32 CFM OpCodes from 64 through 95 and the block of 32 CFM TLV Types from 64 through 95 to the IETF (see Appendix A). This document specifies the IANA considerations for the assignment of values from these two blocks.

IEEE 802.1は（付録A参照）64から95を介して、32のCFMオペコードのブロックとIETF 95を介して64から32のCFM TLVタイプのブロックを割り当てました。この文書では、これら二つのブロックからの値の割り当てのためにIANA問題を指定します。

IEEE 802.1 previously allocated similar blocks of values from 32 through 63 to ITU-T [Y.1731].

IEEE 802.1は、以前にITU-T [Y.1731]に32から63を介して値の同様のブロックを割り当てられました。

3. Security Considerations
3.セキュリティの考慮事項

This document is concerned with assignment of values from the blocks of IEEE 802.1 CFM OpCodes and TLV Types that the IEEE 802.1 Working Group has allocated to the IETF. It is not directly concerned with security.

このドキュメントは、IEEE 802.1 CFMオペコードとIEEE 802.1ワーキンググループはIETFに割り当てられたTLVタイプのブロックからの値の割り当てに関するものです。これは、セキュリティに直接関係はありません。

After considering the referenced liaison from the TRILL WG, IEEE 802.1 has voted to approve the allocation of code points from the Connectivity Fault Management protocol of IEEE Std 802.1Q(TM)-2011 for use by IETF. The expectation of IEEE 802.1 is that these code points will be allocated through IANA only on the basis of IETF standards actions. Specifically, the allocation includes:

TRILL WGから参照連絡を考慮した後、IEEE 802.1は、IETFによる使用のためのIEEE標準802.1Q（TM）-2011の接続障害管理プロトコルからのコードポイントの割り当てを承認するために投票しました。 IEEE 802.1の期待は、これらのコードポイントのみがIETF標準化行動に基づいてIANAによって割り当てられることになるということです。具体的には、割り当てを含みます：

+ 32 CFM OpCode Field values. Reference IEEE Std 802.1Q-2011

+ 32のCFMオペコードフィールドの値。参考IEEE 802.1Q STD-2011

Clause 21.4.3, Table 21-4. The OpCode Field values 64-95 10 are allocated to the IETF.

節21.4.3、表21-4。オペコードフィールド64から95 10はIETFに割り当てられる値。

+ 32 TLV Type Field values. Reference IEEE Std 802.1Q-2011

+ 32のTLVタイプフィールド値。参考IEEE 802.1Q STD-2011

Clause 21.5.1.1, Table 21-6. The Type Field values 64-95 10 are allocated to the IETF.

節21.5.1.1、表21-6。タイプフィールドは、64から95 10はIETFに割り当てられる値。

IEEE Std 802.1Q will be revised at some future date to document this allocation. In the meantime, the allocation will be recorded through the IEEE 802.1 maintenance process.

IEEE標準802.1Qは、この割り当てを文書化するために、いくつかの将来の日付で改訂されます。一方で、割り当てはIEEE 802.1保守プロセスによって記録されます。
