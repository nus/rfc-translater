                           to a New Registry

Abstract

抽象

RFC 5586 generalized the applicability of the pseudowire Associated Channel Header (PW-ACH) into the Generic Associated Channel G-ACh. However, registries and allocations of G-ACh parameters had been distributed throughout different, sometimes unrelated, registries. This document coalesces these into a new "Generic Associated Channel (G-ACh) Parameters" registry under the "Multiprotocol Label Switching Architecture (MPLS)" heading. This document updates RFC 5586.

RFC 5586は、ジェネリック関連するチャンネルG-ACHにスードワイヤ関連するチャネルヘッダ（PW-ACH）の適用性を一般化。しかし、レジストリおよびG-ACHパラメータの割り当ては、時には無関係、異なるレジストリ全体に分散されていました。このドキュメントでは、見出し「アーキテクチャをマルチプロトコル・ラベル・スイッチング（MPLS）」の下に新しい「ジェネリック関連するチャネル（G-ACH）のパラメータ」レジストリにこれらを合体します。この文書は、RFC 5586に更新します。

This document also updates RFCs 6374, 6378, 6427, and 6428.

この資料はまたのRFC 6374、6378、6427、および6428を更新します。

1. Introduction
1.はじめに

RFC 5586 generalized the PW-ACH into the G-ACh. However, registries and allocations of G-ACh namespaces had been distributed throughout different registries. This document coalesces these into a new "Generic Associated Channel (G-ACh) Parameters" registry in the "Multiprotocol Label Switching Architecture (MPLS)" name space. This reorganization achieves two purposes: it allocates the G-ACh registries in their natural place in the MPLS names space, and it provides a single view of the G-ACh registries, to simplify future assignments and avoid potential conflicts. This is an update to RFC 5586 [RFC5586].

RFC 5586は、G-ACHにPW-ACHを一般化。しかし、G-ACH名前空間のレジストリと配分は、異なるレジストリ全体に分散されていました。この文書では、名前空間「アーキテクチャ（MPLS）をマルチプロトコルラベルスイッチング」に新しい「ジェネリック関連するチャネル（G-ACH）のパラメータ」レジストリにこれらを合体します。この再編は2つの目的を達成し：それはMPLSの名前空間での彼らの自然な場所にG-AChのレジストリを割り当て、それは将来の割り当てを簡素化し、潜在的な競合を避けるために、G-AChのレジストリの単一のビューを提供します。これは、RFC 5586 [RFC5586]に更新します。

Further, the "Pseudowire Associated Channel Types" registry is renamed to "Generalized Associated Channel (G-ACh) Types (including Pseudowire Associated Channel Types)" to make its generalized status explicit, and it is moved into the newly created registry.

さらに、「擬似回線関連するチャネルタイプ」レジストリは、その一般的な状況を明示するために「（擬似回線関連するチャネルタイプを含む）一般関連するチャネル（G-ACH）型」に名前が変更され、それは、新しく作成されたレジストリに移動されます。

Additionally, RFC 6374 [RFC6374], RFC 6378 [RFC6378], RFC 6427 [RFC6427], and RFC 6428 [RFC6428] specify allocations within the G-ACh that are now moved into the new registry.

また、RFC 6374 [RFC6374]、RFC 6378 [RFC6378]、RFC 6427 [RFC6427]、およびRFC 6428 [RFC6428]は、新しいレジストリに移動されているG-ACH内の割り当てを指定します。

With respect to where to find these IANA registries, the RFCs listed above are updated as indicated in Section 3; however, the registries themselves are not changed (with the exception of one being renamed). They are moved unchanged to the new registry.

これらIANAレジストリを検索する場所に関しては、上記のRFCはセクション3に示されるように更新されます。しかし、自分自身が変更されていないレジストリ（1を除いて名前が変更されています）。彼らは、新しいレジストリにそのまま移動されます。

2. IANA Considerations
2. IANAの考慮事項

IANA has added this document as a reference for each registry that has been moved or renamed as a result of actions requested by this document.

IANAは、この文書で要求されたアクションの結果として移動または名前が変更された各レジストリのための基準として、このドキュメントを追加しました。

IANA has replaced all the relocated registries with pointers to the new URL or with a redirect.

IANAは新しいURLへのポインタまたはリダイレクトして、すべての移転登記を交換しました。

2.1. Creation of a New Generic Associated Channel (G-ACh) Parameters IANA Registry

2.1。新しい汎用関連するチャネル（G-ACH）パラメータIANAレジストリの作成

IANA has created a new "Generic Associated Channel (G-ACh) Parameters" registry. This is the common registry that will include all the registries being moved in Sections 2.2 and 2.3.

IANAは、新たな「ジェネリック関連するチャネル（G-ACH）のパラメータ」レジストリを作成しました。これは、セクション2.2および2.3に移動されているすべてのレジストリが含まれる一般的なレジストリです。

2.2. Renaming and Moving the Pseudowire Associated Channel Types Registry

2.2。擬似回線関連するチャネルタイプレジストリの名前を変更して移動します

This document renames the "Pseudowire Associated Channel Types" registry [IANA-PWE3] to "MPLS Generalized Associated Channel (G-ACh) Types (including Pseudowire Associated Channel Types)". This registry has been moved and included in the "Generic Associated Channel (G-ACh) Parameters" registry created in Section 2.1 because any additional registries are dependent upon the Associated Channel Header Type.

この文書は、「擬似回線関連するチャネルタイプ」レジストリ[IANA-PWE3]「（擬似回線関連するチャネルタイプを含む）MPLS一般関連するチャネル（G-ACH）タイプ」に名前を変更します。このレジストリは、移動し、任意の追加のレジストリは関連するチャンネルヘッダタイプに依存しているので、2.1節で作成した「ジェネリック関連するチャネル（G-ACH）のパラメータ」レジストリに含まれています。

At the time of publishing this document and moving the registry, the following RFCs have G-ACh Types allocated: [RFC4385], [RFC5586], [RFC5718], [RFC5885], [RFC6374], [RFC6378], [RFC6426], [RFC6427], [RFC6428], [RFC6435], [RFC6478], and [RFC6671].

この文書を公開し、レジストリを移動させる時に、次のRFCは、G-ACHタイプが割り当てられている：[RFC4385]、[RFC5586]、[RFC5718]、[RFC5885]、[RFC6374]、[RFC6378]、[RFC6426]、 [RFC6427]、[RFC6428]、[RFC6435]、[RFC6478]及び[RFC6671]。

[IANA-MPLS-OAM]内のすべてのサブレジストリことに注意することは、「マルチプロトコルラベルスイッチング（MPLS）運用、管理、および管理（OAM）パラメータ」レジストリから削除されました。したがって、IANAはこのドキュメントごとに、MPLS OAMレジストリを削除しています。

3. RFC Updates
3. RFCアップデート

This document updates [RFC5586] renaming the "Pseudowire Associated Channel Types" [IANA-PWE3] to "MPLS Generalized Associated Channel (G-ACh) Types (including Pseudowire Associated Channel Types)".

このドキュメントの更新[RFC5586]「擬似回線関連するチャネルタイプ」[IANA-PWE3]「（擬似回線関連するチャネルタイプを含む）MPLS一般関連するチャネル（G-ACH）タイプ」に名前を変更。

This document also updates the following RFCs by moving the G-ACh related registries to a common "MPLS Generic Associated Channel (G-ACh) Parameters" registry: RFCs 6374, 6378, 6427, and 6428.

RFC 6374、6378、6427、および6428：この文書は、一般的な「MPLSジェネリック関連するチャネル（G-ACH）のパラメータ」レジストリにG-AChの関連レジストリを移動することによって、次のRFCを更新します。

All the registries listed above are moved without any changes to their content. The reason to move them is to create on single place where it is possible to find all the G-ACh parameters.

上記のすべてのレジストリは、その内容を変更することなく移動されます。それらを移動する理由は、すべてのG-AChのパラメータを求めることができる単一の場所に作成することです。
