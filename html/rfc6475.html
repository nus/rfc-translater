             Proxy Mobile IPv6 Management Information Base

Abstract

抽象

This memo defines a portion of the Proxy Mobile IPv6 Management Information Base (MIB) for use with network management protocols in the Internet community. In particular, the Proxy Mobile IPv6 MIB can be used to monitor and control the mobile access gateway (MAG) and the local mobility anchor (LMA) functions of a Proxy Mobile IPv6 (PMIPv6) entity.

このメモは、インターネットコミュニティでのネットワーク管理プロトコルで使用するためにプロキシモバイルIPv6の管理情報ベース（MIB）の一部を定義します。具体的には、プロキシ・モバイルIPv6 MIBを監視し、モバイルアクセスゲートウェイ（MAG）およびプロキシモバイルIPv6（PMIPv6の）エンティティのローカル・モビリティ・アンカー（LMA）の機能を制御するために使用することができます。

2. Overview
2.概要
2.1. The Proxy Mobile IPv6 Protocol Entities
2.1。プロキシモバイルIPv6のプロトコルエンティティ

Proxy Mobile IPv6 (PMIPv6) [RFC5213] is an extension to the Mobile IPv6 (MIPv6) protocol that facilitates network-based localized mobility management (NETLMM) for IPv6 nodes in a PMIPv6 domain. There are three types of entities envisaged by the PMIPv6 protocol.

プロキシモバイルIPv6（PMIPv6の）[RFC5213]はPMIPv6ドメイン内のIPv6ノードのためのネットワークベース局所モビリティ管理（NETLMM）を容易にするモバイルIPv6（MIPv6の）プロトコルの拡張です。 PMIPv6のプロトコルによって想定される事業体の3つのタイプがあります。

mobile node (MN): In the PMIPv6 context, this term is used to refer to an IP host or router whose mobility is managed by the network.

モバイルノード（MN）は：PMIPv6のコンテキストでは、この用語は、そのモビリティネットワークによって管理されているIPホストまたはルータを参照するために使用されます。

local mobility anchor (LMA): Local Mobility Anchor is the home agent for the mobile node in a Proxy Mobile IPv6 domain. It is the topological anchor point for the mobile node's home network prefix(es) and is the entity that manages the mobile node's binding state. The local mobility anchor has the functional capabilities of a home agent as defined in the Mobile IPv6 base specification [RFC6275] with the additional capabilities required for supporting the Proxy Mobile IPv6 protocol as defined in the PMIPv6 specification [RFC5213].

ローカル・モビリティ・アンカー（LMA）：ローカルモビリティアンカーはProxyのモバイルIPv6ドメインのモバイルノードのホームエージェントです。これは、モバイルノードのホームネットワーク接頭語（es）のためのトポロジカルアンカーポイントであり、移動ノードの結合状態を管理するエンティティです。ローカル・モビリティ・アンカーは、PMIPv6の仕様[RFC5213]で定義されるように、プロキシモバイルIPv6プロトコルをサポートするために必要な追加機能を備えたモバイルIPv6の基本仕様[RFC6275]で定義されるように、ホームエージェントの機能的能力を有しています。

mobile access gateway (MAG): Mobile Access Gateway is the entity on an access router that manages the mobility-related signaling for a mobile node that is attached to its access link. It is responsible for tracking the mobile node's movements to and from the access link and for signaling the mobile node's local mobility anchor.

モバイルアクセスゲートウェイ（MAG）：モバイル・アクセス・ゲートウェイは、アクセスリンクに接続されているモバイルノードのためのモビリティ関連シグナリングを管理するアクセスルータのエンティティです。これは、アクセスリンクにしてから、移動ノードの動きを追跡するための、モバイルノードのローカル・モビリティ・アンカーを知らせるための責任があります。

This document defines a set of managed objects (MOs) that can be used to monitor and control PMIPv6 entities.

この文書では、PMIPv6のエンティティを監視および制御するために使用することができる管理対象オブジェクト（MOS）のセットを定義します。

4. MIB Design
4. MIBデザイン

The basic principle has been to keep the MIB as simple as possible and, at the same time, to make it effective enough so that the essential needs of monitoring and control are met.

基本的な原理は、監視と制御の基本的なニーズが満たされるように、十分な、それを有効にするために、同時に、できるだけシンプルなMIBを維持してきました。

The Proxy Mobile IPv6 Management Information Base (PMIPV6-MIB) extends the Mobile IPv6 Management Information Base (MIPV6-MIB) [RFC4295]. It is assumed that PMIPV6-MIB will always be implemented in conjunction with the MOBILEIPV6-MIB [RFC4295]. The PMIPV6-MIB uses the textual conventions defined in the INET-ADDRESS-MIB [RFC4001] and IP-MIB [RFC4293].

プロキシモバイルIPv6の管理情報ベース（のPMIPv6-MIB）が延びているモバイルIPv6管理情報ベース（MIPV6-MIB）[RFC4295]。のPMIPv6-MIBは、常にモバイルIPv6-MIB [RFC4295]に関連して実施されることが想定されます。 PMIPv6の-MIBは、INET-ADDRESS-MIB [RFC4001]とIP-MIB [RFC4293]で定義されたテキストの表記法を使用しています。

The PMIPV6-MIB is composed of the following groups of definitions:

PMIPv6の-MIBは定義の次のグループで構成されています。

- pmip6Core: a generic group containing objects that are common to all the Proxy Mobile IPv6 entities. Objects belonging to this group will be implemented on the corresponding Proxy Mobile IPv6 entity. pmip6BindingCacheTable belongs to this group.

- pmip6Core：すべてのプロキシモバイルIPv6のエンティティに共通のオブジェクトを含む汎用グループ。このグループに属するオブジェクトは、対応するプロキシモバイルIPv6のエンティティに実装されます。 pmip6BindingCacheTableはこのグループに属しています。

- pmip6Mag: this group models the mobile access gateway service. Objects belonging to this group have the "pmip6Mag" prefix and will be implemented on the corresponding MAG.

- pmip6Mag：このグループのモデルモバイルアクセスゲートウェイサービス。このグループに属するオブジェクトは、「pmip6Mag」接頭辞を持っているし、対応するMAGに実装されます。

- pmip6Lma: this group models the local mobility anchor service. Objects belonging to this group have the "pmip6Lma" prefix and will be implemented on the corresponding LMA.

- pmip6Lma：このグループのモデルローカル・モビリティ・アンカーサービス。このグループに属するオブジェクトは、「pmip6Lma」接頭辞を持っており、対応するLMAに実装されます。

- pmip6Notifications: defines the set of notifications that will be used to asynchronously monitor the Proxy Mobile IPv6 entities.

- pmip6Notifications：非同期的にプロキシモバイルIPv6のエンティティを監視するために使用される通知のセットを定義します。

The tables contained in the above groups are as follows:

次のように上記の基に含まれるテーブルは、次のとおりです。

- pmip6BindingCacheTable: models the Binding Cache on the local mobility anchor.

- pmip6BindingCacheTable：ローカルモビリティアンカーのモデルバインディングキャッシュ。

- pmip6MagProxyCOATable: models the Proxy Care-of Addresses configured on the egress interfaces of the mobile access gateway.

- pmip6MagProxyCOATable：モデルプロキシ気付アドレスは、モバイル・アクセス・ゲートウェイの出力インターフェイスに構成されています。

- pmip6MagMnIdentifierTable: provides a mapping from the MAG-internal pmip6MagMnIndex to the mobile node identifier.

- pmip6MagMnIdentifierTable：モバイルノード識別子にMAG-内部pmip6MagMnIndexからのマッピングを提供します。

- pmip6MagMnLLIdentifierTable: provides a mapping from the MAG-internal pmip6MagMnLLIndex to the corresponding interface of the mobile node link-layer identifier.

- pmip6MagMnLLIdentifierTable：モバイルノードのリンク層識別子の対応するインターフェースにMAG-内部pmip6MagMnLLIndexからのマッピングを提供します。

- pmip6MagHomeNetworkPrefixTable: contains the home network prefixes assigned to interfaces of all mobile nodes attached to the MAG. Each interface is distinguished by the attached mobile node identifier (MN-Identifier) and the link-layer identifier (MN-LL-Identifier).

- pmip6MagHomeNetworkPrefixTable：MAGに接続されているすべての移動ノードのインタフェースに割り当てられたホーム・ネットワーク・プレフィクスを含んでいます。各インターフェースは、添付のモバイルノード識別子（MN-識別子）とリンク層識別子（MN-LL-識別子）によって区別されます。

- pmip6MagBLTable: models the Binding Update List (BL) that includes PMIPv6-related information and is maintained by the mobile access gateway.

- pmip6MagBLTable：モデルバインディング更新リスト（BL）のPMIPv6に関する情報を含み、モバイル・アクセス・ゲートウェイによって保持されています。

- pmip6MagMnProfileTable: contains the mobile node's policy profile that includes the essential operational parameters that are required by the network entities for managing the mobile node's mobility service.

- pmip6MagMnProfileTable：モバイルノードのモビリティ・サービスを管理するためのネットワーク・エンティティによって必要とされる基本的な動作パラメータを含んでいるモバイルノードのポリシープロファイルが含まれています。

- pmip6LmaLMAATable: contains the LMA Addresses (LMAAs) that are configured on the local mobility anchor. Each LMA Address acts as a transport endpoint of the tunnel between the local mobility anchor and the mobile access gateway.

- pmip6LmaLMAATable：ローカルモビリティアンカーに設定されているLMAアドレス（LMAAs）が含まれています。各LMAアドレスは、ローカル・モビリティ・アンカーとモバイルアクセスゲートウェイとの間のトンネルのトランスポートエンドポイントとして機能します。

- pmip6LmaMnIdentifierTable: provides a mapping from the LMA-internal pmip6BindingMnIndex to the mobile node identifier.

- pmip6LmaMnIdentifierTable：モバイルノード識別子にLMA-内部pmip6BindingMnIndexからのマッピングを提供します。

- pmip6LmaMnLLIdentifierTable: provides a mapping from the LMA-internal pmip6BindingMnLLIndex to the corresponding interface of the mobile node link-layer identifier.

- pmip6LmaMnLLIdentifierTable：モバイルノードのリンク層識別子の対応するインターフェースにLMA-内部pmip6BindingMnLLIndexからのマッピングを提供します。

- pmip6LmaHomeNetworkPrefixTable: contains the list of home network prefixes assigned to the connected interfaces of the mobile nodes anchored on an LMA.

- pmip6LmaHomeNetworkPrefixTable：LMAに固定モバイルノードの接続インタフェースに割り当てられたホーム・ネットワーク・プレフィクスのリストを含みます。
