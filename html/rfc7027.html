Abstract

抽象

This document specifies the use of several Elliptic Curve Cryptography (ECC) Brainpool curves for authentication and key exchange in the Transport Layer Security (TLS) protocol.

この文書では、認証およびTransport Layer Security（TLS）プロトコルにおける鍵交換のためのいくつかの楕円曲線暗号（ECC）Brainpool曲線の使用を指定します。

1. Introduction
1. はじめに

[RFC5639] specifies a new set of elliptic curve groups over finite prime fields for use in cryptographic applications. These groups, denoted as ECC Brainpool curves, were generated in a verifiably pseudo-random way and comply with the security requirements of relevant standards from ISO [ISO1] [ISO2], ANSI [ANSI1], NIST [FIPS], and SecG [SEC2].

[RFC5639]は暗号アプリケーションで使用するための有限素体上の楕円曲線グループの新しいセットを指定します。 ECC Brainpool曲線として示されるこれらの基は、検証可能擬似ランダムな方法で生成され、ISO [ISO1] [ISO2]、ANSI [ANSI1]、NIST [FIPS]とはSecG [SEC2から関連規格のセキュリティ要件に準拠しました]。

[RFC4492] defines the usage of elliptic curves for authentication and key agreement in TLS 1.0 and TLS 1.1; these mechanisms may also be used with TLS 1.2 [RFC5246]. While the ASN.1 object identifiers defined in [RFC5639] already allow usage of the ECC Brainpool curves for TLS (client or server) authentication through reference in X.509 certificates according to [RFC3279] and [RFC5480], their negotiation for key exchange according to [RFC4492] requires the definition and assignment of additional NamedCurve IDs. This document specifies such values for three curves from [RFC5639].

[RFC4492]はTLS 1.0およびTLS 1.1における認証および鍵合意のために楕円曲線の使用を定義します。これらの機構はまた、TLS 1.2 [RFC5246]で使用することができます。 [RFC5639]で定義されたASN.1オブジェクト識別子が既にTLS（クライアントまたはサーバ）のためのECC Brainpool曲線の使用を可能にしながら、認証X.509証明書における参照スルー[RFC3279]及び[RFC5480]によれば、鍵交換のためのそれらのネゴシエーション[RFC4492]によれば、追加NamedCurve IDの定義および割り当てを必要とします。このドキュメントは[RFC5639]からの3つの曲線のためのそのような値を指定します。

2. Brainpool NamedCurve Types
2. Brainpool NamedCurveタイプ

According to [RFC4492], the name space NamedCurve is used for the negotiation of elliptic curve groups for key exchange during a handshake starting a new TLS session. This document adds new NamedCurve types to three elliptic curves defined in [RFC5639] as follows:

[RFC4492]によると、名前空間NamedCurveは、新たなTLSセッションを開始ハンドシェーク中に鍵交換のための楕円曲線グループの交渉のために使用されています。この文書では次のように[RFC5639]で定義された3本の楕円曲線に新しいNamedCurveタイプを追加します。 

           enum {
                brainpoolP256r1(26),
                brainpoolP384r1(27),
                brainpoolP512r1(28)
           } NamedCurve;

These curves are suitable for use with Datagram TLS [RFC6347].

これらの曲線は、データグラムTLS [RFC6347]での使用に適しています。

Test vectors for a Diffie-Hellman key exchange using these elliptic curves are provided in Appendix A.

これらの楕円曲線を用​​いたDiffie-Hellman鍵交換のためのテストベクトルは、付録Aに設けられています。

4. Security Considerations
4.セキュリティについての考慮事項

The security considerations of [RFC5246] apply to the ECC Brainpool curves described in this document.

[RFC5246]のセキュリティ上の考慮事項は、この文書で説明するECC Brainpool曲線に適用されます。

The confidentiality, authenticity, and integrity of the TLS communication is limited by the weakest cryptographic primitive applied. In order to achieve a maximum security level when using one of the elliptic curves from Table 1 for authentication and/or key exchange in TLS, the key derivation function; the algorithms and key lengths of symmetric encryption; and message authentication (as well as the algorithm, bit length, and hash function used for signature generation) should be chosen according to the recommendations of [NIST800-57] and [RFC5639]. Furthermore, the private Diffie-Hellman keys should be selected with the same bit length as the order of the group generated by the base point G and with approximately maximum entropy.

機密性、信頼性、およびTLS通信の整合性が適用される最も弱い暗号プリミティブによって制限されています。認証および/またはTLS、鍵導出関数に鍵交換のための表1から楕円曲線のいずれかを使用したときに最大のセキュリティレベルを達成するために。アルゴリズムおよび対称暗号化の鍵長、メッセージ認証（並びにアルゴリズム、ビット長、および署名生成のために使用されるハッシュ関数）の勧告[NIST800-57]及び[RFC5639]に従って選択されるべきです。また、民間のDiffie-Hellmanキーは、ベースポイントGによって約最大エントロピーを用いて生成されたグループの順序と同じビット長で選択されるべきです。

Implementations of elliptic curve cryptography for TLS may be susceptible to side-channel attacks. Particular care should be taken for implementations that internally transform curve points to points on the corresponding "twisted curve", using the map (x',y') = (x*Z^2, y*Z^3) with the coefficient Z specified for that curve in [RFC5639], in order to take advantage of an efficient arithmetic based on the twisted curve's special parameters (A = -3). Although the twisted curve itself offers the same level of security as the corresponding random curve (through mathematical equivalence), an arithmetic based on small curve parameters may be harder to protect against side- channel attacks. General guidance on resistance of elliptic curve cryptography implementations against side-channel-attacks is given in [BSI1] and [HMV].

TLS用の楕円曲線暗号の実装では、サイドチャネル攻撃を受けやすいかもしれません。特定のケアは、内部係数Zに対応するマップを用いて、「ねじれた曲線」、（X「Y」）=（Xの*のZ ^ 2、Yの*のZ ^ 3）上の点にカーブポイント変換の実装のために取らなければなりませんツイスト曲線の特別なパラメータに基づいて効率的な演算を利用するために、[RFC5639]にその曲線のために指定された（A = -3）。ツイスト曲線自体が（数学的等価介して）対応するランダム曲線と同じレベルのセキュリティを提供するが、小さな曲線パラメータに基づく算術側 - チャネル攻撃から保護するために困難であってもよいです。サイドチャネル攻撃に対する楕円曲線暗号の実装の抵抗の一般的なガイダンスは、[HMV] [BSI1]で与えられています。
