Abstract

抽象

This document describes a Uniform Resource Name (URN) namespace for the Hybrid Broadcast Broadband TV (HbbTV) Association for naming persistent resources defined within HbbTV specifications. Example resources include technical documents and specifications, Extensible Markup Language (XML) Schemas, classification schemes, XML Document Type Definitions (DTDs), namespaces, style sheets, media assets, and other types of resources produced or managed by HbbTV.

この文書では、HbbTV規格内で定義された永続的なリソースを命名するためのハイブリッド放送ブロードバンドTV（HbbTV）協会統一リソース名（URN）名前空間について説明します。例のリソースは、技術文書や仕様、拡張マークアップ言語（XML）スキーマ、分類スキーム、XML文書型定義（DTDの）、名前空間、スタイルシート、メディア資産、およびHbbTVによって生成または管理するリソースの他の種類が含まれます。

1. Introduction
1.はじめに

HbbTV (Hybrid Broadcast Broadband TV) is a new industry standard providing an open and business-neutral technology platform that seamlessly combines TV services delivered via broadcast with services delivered via broadband and also enables access to Internet-only services for consumers using connected TVs and set-top boxes. The founding members of the HbbTV consortium together with a large group of supporters jointly developed the HbbTV specification to create a global standard for hybrid entertainment services. Version 1.2.1 of this specification was published by ETSI as ETSI TS 102 796 [HbbTV12] in November 2012.

HbbTV（ハイブリッド放送ブロードバンドTV）は、シームレスにブロードバンドを経由して提供されるサービスで放送経由で配信TVサービスを組み合わせたオープンし、ビジネス・中立的技術プラットフォームを提供する、新しい業界標準であり、また、接続テレビ、セットを使用して消費者のためのインターネット専用サービスへのアクセスを可能にします-topボックス。サポーターの大規模なグループと一緒にHbbTVコンソーシアムの設立メンバーは共同でハイブリッドエンターテインメント・サービスのグローバルスタンダードを作成するために、HbbTV規格を開発しました。この仕様のバージョン1.2.1は2012年11月に[HbbTV12] 102 796 ETSI TSとしてETSIにより出版されました。

The HbbTV specification is based on existing standards and web technologies including the Open IPTV Forum (OIPF), Consumer Electronics Association (CEA), Digital Video Broadcasting (DVB), and World Wide Web Consortium (W3C). The standard provides the features and functionality required to deliver feature-rich broadcast and Internet services. Utilizing standard Internet technology, it enables rapid application development. It defines minimum requirements simplifying the implementation in devices and leaving room for differentiation; this limits the investment required by consumer electronics manufacturers to build compliant devices. The HbbTV Association is a pan-European initiative aimed at providing an alternative to proprietary technologies and delivering an open platform for broadcasters to deliver value-added on-demand services to the end consumer.

HbbTV仕様は既存の標準とオープンIPTVフォーラム（OIPF）、コンシューマー・エレクトロニクス協会（CEA）、デジタルビデオ放送（DVB）、およびWorld Wide Webコンソーシアム（W3C）を含むWeb技術に基づいています。標準では、豊富な機能を備えた放送とインターネットサービスを提供するために必要な機能と機能を提供します。標準のインターネット技術を活用し、それが迅速なアプリケーション開発を可能にします。これは、最小要件デバイスでの実装を簡素化し、差別化の余地を残しを定義します。これは、対応デバイスを構築するために家電メーカーによって必要な投資を制限します。 HbbTV協会は、独自の技術に代わるものを提供し、放送局が最終消費者に付加価値のオンデマンドサービスを提供するためのオープンプラットフォームを提供を目的とした汎欧州イニシアチブです。

HbbTV has a wide range of supporters from across the broadcaster and consumer electronics industries.

HbbTVは、放送業界と家電業界全体からサポーターの広い範囲を持っています。

HbbTV would like to assign unique, permanent, location-independent names based on URNs for some resources it produces or manages. These URNs will be constructed according to the URN syntax defined in [RFC2141]. This namespace specification is for a formal namespace to be registered according to the procedures set forth in [RFC3406].

HbbTVは、それが生成または管理していくつかのリソースのためにURNをもとに独自の、永続的な、場所に依存しない名前を割り当てたいです。これらのURNは、[RFC2141]で定義されたURN構文に従って構築されます。この名前空間仕様は[RFC3406]に記載された手順に従って登録する正式な名前空間のためのものです。

4. Namespace Considerations
4.名前空間の考慮事項

A unique formal namespace is required by HbbTV in order to specify how the various existing standards can be linked in order to create a true end-to-end ecosystem for standards-based IPTV deployments and to provide the necessary system-wide resources.

独特の正式な名前空間は、様々な既存の規格は、標準ベースのIPTVの展開のための真のエンド・ツー・エンドの生態系を作成するために、必要なシステム全体のリソースを提供するためにリンクすることができる方法を指定するために、HbbTVにより必要とされます。

5. Community Considerations
5.コミュニティの考慮事項

6. Security Considerations
6.セキュリティの考慮事項

There are no additional security considerations other than those normally associated with the use and resolution of URNs in general, which are described in [RFC1737], [RFC2141], and [RFC3406].

[RFC1737]に記載されている通常、一般的にのURNの使用および解像度に関連するもの、[RFC2141]及び[RFC3406]以外の付加的なセキュリティ問題がありません。

This document registers a namespace for URNs. HbbTV may assign special meaning to certain of the characters of the Namespace Specific String (NSS) in its specifications. Any security consideration resulting from such assignment is outside the scope of this document.

この文書では、URNのための名前空間を登録します。 HbbTVは、その仕様の名前空間固有の文字列（NSS）の文字の特定のに特別な意味を割り当てることができます。そのような割り当てから得られる任意のセキュリティ上の考慮事項は、この文書の範囲外です。
