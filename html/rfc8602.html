1. Introduction
1. はじめに

This memo updates the IANA registry rules for the Telephony Routing over IP (TRIP) protocol, by no longer requiring that postal addresses be included in contact information. No use for the addresses is identified, and omitting this information provides a privacy benefit.

このメモは、住所情報を連絡先情報に含める必要がなくなるため、IP（TRIP）プロトコルを介したテレフォニールーティングのIANAレジストリルールを更新します。 アドレスの使用は特定されていないため、この情報を省略するとプライバシーが保護されます。

This memo updates RFC 3219.

このメモはRFC 3219を更新します。

2. IANA Considerations
2. IANAの考慮事項

This memo updates the registration rules [RFC8126] for TRIP Attributes ([RFC3219], Section 13.2) and TRIP IP Telephony Administrative Domain (ITAD) Numbers ([RFC3219], Section 13.5) as follows.

このメモは、TRIP属性（[RFC3219]、セクション13.2）およびTRIP IPテレフォニー管理ドメイン（ITAD）番号（[RFC3219]、セクション13.5）の登録規則[RFC8126]を次のように更新します。

The IETF no longer requires IANA to collect postal address information when adding TRIP registry entries. Additionally, IANA has removed previously collected postal addresses from the registries.

IETFでは、TRIPレジストリエントリを追加するときに、IANAが郵便アドレス情報を収集する必要がなくなりました。 さらに、IANAは以前に収集した住所をレジストリから削除しました。

3. Security Considerations
3. セキュリティに関する考慮事項

No security impacts of this change have been identified.

この変更によるセキュリティへの影響は確認されていません。
