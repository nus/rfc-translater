Abstract

抽象

This memo defines a registry for IP Performance Metrics (IPPM). It assigns and registers an initial set of OBJECT IDENTITIES to currently defined metrics in the IETF.

このメモはIPパフォーマンス・メトリック（IPPM）のレジストリを定義します。それは割り当て、現在IETFのメトリックを定義するためのオブジェクトIDの初期セットを登録します。

This memo also defines the rules for adding IP Performance Metrics that are defined in the future and for encouraging all IP performance metrics to be registered here.

また、このメモは、将来に定義されているIPパフォーマンス・メトリックを追加するために、ここで登録するすべてのIPパフォーマンス測定基準を奨励するための規則を定義します。

IANA has been assigned to administer this new registry.

IANAは、この新しいレジストリを管理するために割り当てられています。

1. The Internet-Standard Management Framework
1.インターネット標準管理フレームワーク

For a detailed overview of the documents that describe the current Internet-Standard Management Framework, please refer to section 7 of RFC 3410 [RFC3410].

現在のインターネット標準の管理フレームワークを記述したドキュメントの詳細な概要については、RFC 3410 [RFC3410]のセクション7を参照してください。

Managed objects are accessed via a virtual information store, termed the Management Information Base or MIB. MIB objects are generally accessed through the Simple Network Management Protocol (SNMP). Objects in the MIB are defined using the mechanisms defined in the Structure of Management Information (SMI). This memo specifies a MIB module that is compliant to the SMIv2, which is described in STD 58, RFC 2578 [RFC2578], STD 58, RFC 2579 [RFC2579] and STD 58, RFC 2580 [RFC2580].

管理対象オブジェクトが仮想情報店を介してアクセスされ、管理情報ベースまたはMIBと呼ばれます。 MIBオブジェクトは、一般的に簡易ネットワーク管理プロトコル（SNMP）を介してアクセスされます。 MIBのオブジェクトは、管理情報（SMI）の構造で定義されたメカニズムを使用して定義されています。このメモは、STD 58、RFC 2578 [RFC2578]、STD 58、RFC 2579 [RFC2579]とSTD 58、RFC 2580 [RFC2580]に記載されているSMIv2のに準拠しているMIBモジュールを指定します。

2. Overview
2.概要

This memo defines a registry of the current metrics and a framework for the integration of future metrics for the following reasons:

このメモは、現在のメトリックと、次の理由により、将来のメトリックの統合のための枠組みのレジストリを定義します。

o to permit metrics to be clearly referenced by MIB modules or other data models;

はっきりMIBモジュールまたは他のデータモデルによって参照されるメトリックを可能にするために、O。

o to provide metrics identifiers for measurement interoperability;

oは、測定の相互運用性のためのメトリクス識別子を提供すること。

o to take special care with the integration of future standardized metrics because it is a continuous process;

それは連続的なプロセスであるため、将来の標準化されたメトリクスの統合に特別な世話をするために、O。

o to provide room where other standards bodies can register their metrics in order to encourage IP performance metrics to have OBJECT IDENTITIES rooted at a common point because the intent of the IPPM WG is to cooperate with other appropriate standards bodies (or fora) to promote consistent metrics;

一貫性を促進するためにIPPM WGの目的は、他の適切な標準化団体（またはフォーラム）と協力することであるため、他の標準化団体が、共通点をルートとOBJECT IDを持っているIPパフォーマンス・メトリックを奨励するために、そのメトリックを登録することができます部屋を提供するために、Oメトリクス。

o and, similarly, to provide room for enterprises to register metrics.

企業はメトリックを登録するためのOと、同様に、部屋を提供します。

3. IP Performance Metrics Registry Framework
3. IPパフォーマンス・メトリックレジストリのフレームワーク

MIB modules need to be able to reference IPPM Metrics precisely. The registry associates an OBJECT-IDENTITY with each metric. For example, Type-P-One-way-Delay and Type-P-One-way-Delay-Poisson-Stream have different OBJECT IDENTITIES.

MIBモジュールは、正確にIPPMメトリックを参照できるようにする必要があります。レジストリは、各メトリックとOBJECT-IDENTITYを関連付けます。例えば、タイプP-ワンウェイ遅延とType-P-ワンウェイ遅延ポアソンストリームは、異なるオブジェクトIDを持っています。

The registry has one branch. Metrics are identified in the 'ianaIppmMetrics' subtree.

レジストリは、一方の分岐があります。メトリックは、「ianaIppmMetrics」サブツリー内で識別されています。

This document defines an initial registry of the existing metrics defined in the IPPM WG and the rules to manage the registry.

この文書では、レジストリを管理するためにIPPM WGとルールで定義されている既存のメトリックの初期登録を定義しています。

Documents defining metrics in the future will include in the IANA section the registration information to identify these metrics unambiguously.

将来的に指標を定義する文書は、IANAのセクションで明確にこれらの指標を識別するための登録情報が含まれます。

5. IANA Considerations
5. IANAの考慮事項

This section describes the rules for the management of the registry by IANA.

このセクションでは、IANAによってレジストリの管理のためのルールを説明しています。

5.1. Management Rules
5.1。管理ルール

Registrations are done sequentially by IANA in the ianaIppmMetrics subtree on the basis of 'Specification Required', as defined in [RFC2434].

登録は[RFC2434]で定義されるように、「仕様が必要」に基づいてianaIppmMetricsサブツリー内IANAによって順次行われます。

The reference of the specification must point to a stable document including a title, a revision, and a date.

仕様の参照は、タイトル、改訂、および日付など、安定した文書を指している必要があります。

The name always starts with the name of the organization and must respect the SMIv2 rules for descriptors defined in section 3.1 of [RFC2578].

名前は、常に組織の名前で始まり、[RFC2578]のセクション3.1で定義された記述子のSMIv2のルールを尊重しなければなりません。

A document that creates new metrics would have an "IANA Considerations" section in which it would describe new metrics to be registered.

新しいメトリックを作成した文書は、それが登録される新しいメトリックを記述することになるで「IANAの考慮事項」のセクションを持っているでしょう。

An OBJECT IDENTITY assigned to a metric is definitive and cannot be reused. If a new version of a metric is produced, then it is assigned with a new name and a new identifier.

メトリックに割り当てられたオブジェクトIDは決定的で、再利用することはできません。メトリックの新しいバージョンが作成されている場合、それは新しい名前と新しい識別子が割り当てられています。

7. Security Considerations
7.セキュリティの考慮事項

This module does not define any management objects. Instead, it assigns a set of OBJECT-IDENTITIES which may be used by other MIB modules to identify specific IP Performance Metrics.

このモジュールは、任意の管理オブジェクトを定義していません。代わりに、特定のIPパフォーマンス・メトリックを識別するために、他のMIBモジュールによって使用することができるオブジェクト・アイデンティティのセットを割り当てます。

Meaningful security considerations can only be written in the MIB modules that define management objects. This document has therefore no impact on the security of the Internet.

意味のあるセキュリティ上の考慮事項は、管理オブジェクトを定義するMIBモジュールに書き込むことができます。従って、この文書は、インターネットのセキュリティには影響を与えません。
