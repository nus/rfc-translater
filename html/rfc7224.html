1. Introduction
1.はじめに

This document defines the initial version of the iana-if-type YANG module for interface type definitions.

この文書では、インタフェースタイプ定義のためのIANA-IF型YANGモジュールの初期バージョンを定義します。

The iana-if-type module reflects IANA's existing "ifType definitions" registry [IFTYPE-IANA-REGISTRY]. The latest revision of the module can be obtained from the IANA web site.

IANA-IF型モジュールは、IANAの既存の "ifTypeの定義" レジストリ[ifTypeの-IANA-REGISTRY]を反映しています。モジュールの最新版は、IANAのWebサイトから入手することができます。

Whenever a new interface type is added to the "ifType definitions" registry, the IANAifType-MIB and the iana-if-type YANG module are updated by IANA.

新しいインターフェイスタイプが「ifTypeの定義」レジストリに追加されるたびに、IANAifType-MIBとIANA-IF型YANGモジュールは、IANAによって更新されます。

3. IANA Considerations
3. IANAの考慮事項

This document defines the initial version of the IANA-maintained iana-if-type YANG module.

この文書は、IANAによって維持IANA-IF型YANGモジュールの初期バージョンを定義します。

The iana-if-type YANG module is intended to reflect the "ifType definitions" registry [IFTYPE-IANA-REGISTRY], exactly like the IANAifType-MIB MIB module [IANAifType-MIB] reflects the "ifType definitions" registry.

IANA-IF型YANGモジュールが正確にIANAifType-MIB MIBモジュールのような "ifTypeの定義" レジストリ[ifTypeの-IANA-REGISTRY]、[IANAifType-MIB]を反映することを意図された "ifTypeの定義"、レジストリを反映しています。

IANA has added this new note to the "iana-if-type YANG Module" registry:

IANAは、「IANA-IF型YANGモジュール」レジストリにこの新しいノートを追加しました：

Interface types must not be directly added to the iana-if-type YANG module. They must instead be added to the "ifType definitions" registry.

インターフェイスタイプが直接IANA-IF型YANGモジュールに追加してはなりません。彼らは代わりに「ifTypeの定義」レジストリに追加する必要があります。

When an interface type is added to the "ifType definitions" registry, a new "identity" statement must be added to the iana-if-type YANG module. The name of the "identity" is the same as the corresponding enumeration in the IANAifType-MIB. The following substatements to the "identity" statement should be defined:

インターフェイスタイプが「ifTypeの定義」レジストリに追加されると、新しい「アイデンティティ」の文は、IANA-IF型YANGモジュールに追加する必要があります。 「アイデンティティ」の名前は、IANAifType-MIBの対応する列挙と同じです。 「アイデンティティ」の文を次のようにサブステートメントを定義する必要があります。

"base": Contains the value "iana-interface-type".

「塩基」：「IANA-インターフェース型」の値が含まれています。

"status": Include only if a registration has been deprecated (use the value "deprecated") or obsoleted (use the value "obsolete").

「状態」：登録（「時代遅れ」の値を使用）、非推奨（「非推奨」値を使用）、または廃止された場合にのみ含めます。

"description": Replicate the description from the registry, if any. Insert line breaks as needed so that the line does not exceed 72 characters.

「説明」：いずれかの場合には、レジストリから説明を複製します。必要に応じて行が72文字を超えないように改行を挿入します。

"reference": Replicate the reference from the registry, if any, and add the title of the document.

「参照」：いずれかの場合には、レジストリから参照を複製し、文書のタイトルを追加します。

Unassigned or reserved values are not present in the module.

未割り当てまたは予約値は、モジュール内に存在しません。

When the iana-if-type YANG module is updated, a new "revision" statement must be added in front of the existing revision statements.

IANA-IF型YANGモジュールが更新されると、新しい「リビジョン」の文は、既存のリビジョン文の前に追加する必要があります。

IANA has added this new note to the "ifType definitions" registry:

IANAは、「ifTypeの定義」レジストリにこの新しいノートを追加しました：

When this registry is modified, the YANG module iana-if-type must be updated as defined in RFC 7224.

このレジストリを変更すると、YANGはモジュールIANA型があれば、RFC 7224で定義されているように更新する必要があります。

The "Reference" text in the "ifType definitions" registry has been updated as:

「ifTypeの定義は、」レジストリ内の「参照」のテキストは次のように更新されました。

OLD: [RFC1213][RFC2863]

OLD：[RFC1213] [RFC2863]

NEW: [RFC1213][RFC2863][RFC7224]

NEW：[RFC1213] [RFC2863] [RFC7224]
