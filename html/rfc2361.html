1 Introduction

1はじめに

Internet-oriented multimedia applications reference multimedia content via predefined mechanisms (e.g., [2]). In the general case, this content was created primarily for the use of these Internet-oriented applications. Unfortunately, this Internet-oriented multimedia content represents a small minority of the total amount of multimedia content that has been created to date.

インターネット指向のマルチメディアアプリケーションは、あらかじめ定義された機構を介してマルチメディアコンテンツを参照（例えば、[2]）。一般的なケースでは、このコンテンツは、主にこれらのインターネット指向のアプリケーションを使用するために作成されました。残念ながら、このインターネット指向のマルチメディアコンテンツは、これまでに作成されたマルチメディアコンテンツの総量の少数を表します。

For this reason, a growing interest is forming in establishing mechanisms by which the repertoire of multimedia content available to Internet-oriented applications(e.g., for RTSP [3]) may be greatly extended to include multimedia content that has been created outside of distinctly Internet contexts. For this to occur, a mechanism must be created for Internet protocols (e.g., [1], [3], [4]) to be able to identify the codecs by which this so-called "traditional" multimedia content has been encoded.

この理由のため、関心が確立機構に形成されてインターネット指向のアプリケーションに利用可能なマルチメディアコンテンツのレパートリー（例えば、RTSP [3]）が大きくはっきりインターネットの外部で作成されたマルチメディアコンテンツを含むように拡張することができますコンテキスト。これが発生するため、機構（例えば、[1]、[3]、[4]）このいわゆる「伝統的な」マルチメディアコンテンツが符号化されたことにより、コーデックを識別することができるようにするインターネットプロトコルのために作成されなければなりません。

Unfortunately, several distinct encoding systems exist for traditional multimedia content. Each system has its own registry to ensure unique and stable codec identifications within that system. Perhaps the best known of these registries are Microsoft (for WAVE and AVI content) and Apple (for QuickTime content).

残念ながら、いくつかの異なる符号化方式は、従来のマルチメディアコンテンツのために存在します。各システムは、そのシステム内で一意と安定したコーデックの識別を確実にするために、独自のレジストリを持っています。おそらく最高のこれらのレジストリの既知の（WAVEやAVIコンテンツのための）、マイクロソフトとアップル（QuickTimeコンテンツ用）です。

The purpose of this paper is to establish a mechanism by which codecs registered within Microsoft's WAVE and AVI Registries may be referenced within the IANA Namespace by Internet applications.

本稿の目的は、MicrosoftのWAVEやAVIレジストリ内に登録されるコーデックは、インターネットアプリケーションによってIANA名前空間内で参照されるかもしれないメカニズムを確立することです。

2 References to Registries within the IANA Vendor Tree

IANAベンダーツリー内のレジストリに2つの参照

Reference [7] specifies that the IANA Namespace encompasses several trees. Discussions within the IETF-Types mailing list concluded that the most appropriate tree in which to reference codecs, which had already been registered by non-IANA Registries, is the Vendor Tree.

参考文献[7] IANA名前空間には、いくつかの木を包含することを指定します。メーリングリストIETF-タイプ内の議論は、すでに非IANAレジストリによって登録されていたコーデックを参照するには最も適した木は、ベンダーの木であると結論づけました。

As a result, the non-IANA registry is identified within the IANA Vendor tree by vnd.RegistryName. A specific codec, which has been registered within that registry, is identified by a required codec parameter as specified by Section 2.2.3 of [7].

その結果、非IANAレジストリはvnd.RegistryNameによってIANAベンダーツリー内で識別されます。 [7]のセクション2.2.3で指定されているレジストリ内に登録されている特定のコーデックは、必要なコーデックパラメータによって識別されます。

3 WAVE and AVI Registries

3 WAVEやAVIレジストリ

Both the WAVE and AVI Registries are historic databases that have been maintained by Microsoft as a free service. The Registries sought to assist developers of WAVE and AVI content and to standardize WAVE and AVI content by 1) avoiding conflict and/or duplication with current definitions, and 2) providing the registered information in a standard document and format that is publicly available. The historic nature of these databases implies that unless the original registrants informed the registrar of a change of status (e.g., company acquired, new contact, new location, new phone), the contact information has generally not been updated from the originally registered values.

WAVEやAVIレジストリの両方が無料のサービスとしてMicrosoftによって維持されている歴史的なデータベースです。レジストリは、現在の定義と矛盾及び/又は重複を避ける）WAVEとAVIコンテンツの開発を支援し、1波とAVIコンテンツを標準化しようと、2）公的に入手可能である標準的な文書フォーマットに登録された情報を提供します。これらのデータベースの歴史的な性質は、元の登録者は、ステータスの変更の登記を通知しない限り（例えば、同社が買収、新しい連絡先、新しい場所、新しい携帯電話）、連絡先情報は、一般的に、元々登録された値から更新されていないことを意味します。

4 Mapping Codec IDs to GUID Values

GUID値に4つのマッピングコーデックのID

Direct mappings exist between WAVE Format IDs and GUIDs and between FourCC codec values and GUIDs [5]. [Note: GUIDs are Globally Unique Identifiers that are also known as Universally Unique Identifiers (UUIDs). UUIDs have been standardized within the Open Software Foundation's (OSF) Distributed Computing Environment (DCE).] These mappings enable GUID-oriented software to directly refer to these historic codec values. For example, the Advanced Streaming Format (ASF) [6] uses GUID values to refer to codecs, and the following mechanism is used to convert the historic WAVE and AVI codec values into the appropriate GUID value for use within ASF.

直接マッピングは、WAVE形式のIDとGUIDの間とするFourCCコーデックの値とGUID [5]の間に存在します。 [注：GUIDはまた、汎用一意識別子（UUIDの）として知られているグローバル一意識別子です。 UUIDがオープンSoftware Foundationの（OSF）分散コンピューティング環境（DCE）の中に標準化されている。これらのマッピングは、直接これらの歴史的なコーデックの値を参照するGUID指向ソフトウェアを有効にします。例えば、アドバンストストリーミングフォーマット（ASF）[6]コーデックを参照するGUID値を使用し、以下のメカニズムがASF内で使用するための適切なGUID値に歴史的な波とAVIコーデック値を変換するために使用されます。
