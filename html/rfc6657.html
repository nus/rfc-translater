1. Introduction and Overview
1.はじめにと概要 
   RFC 2046 specified that the default "charset" parameter (i.e., the
   value used when the parameter is not specified) is "US-ASCII"
   (Section 4.1.2 of [RFC2046]).  RFC 2616 changed the default for use
   by HTTP (Hypertext Transfer Protocol) to be "ISO-8859-1" (Section
   3.7.1 of [RFC2616]).  This encoding is not very common for new
   "text/*" media types and a special rule in the HTTP specification
   adds confusion about which specification ([RFC2046] or [RFC2616]) is
   authoritative in regards to the default charset for "text/*" media
   types.

Many complex text subtypes such as "text/html" [RFC2854] and "text/ xml" [RFC3023] have internal (to their format) means of describing the charset. Many existing User Agents ignore the default of "US-ASCII" rule for at least "text/html" and "text/xml".

そのような「テキスト/ HTML」[RFC2854]及び「テキスト/ XML」[RFC3023]などの多くの複雑なテキストサブタイプは、（それらのフォーマットに）内部の文字セットを記述する手段を有します。多くの既存のユーザエージェントは、少なくとも「text / html」および「text / xmlで」の「US-ASCII」ルールのデフォルトを無視します。 

   This document changes RFC 2046 rules regarding default "charset"
   parameter values for "text/*" media types to better align with common
   usage by existing clients and servers.  It does not change the
   defaults for any currently registered media type.

3.  New Rules for Default "charset" Parameter Values for "text/*" Media
    Types

Section 4.1.2 of [RFC2046] says:

[RFC2046]のセクション4.1.2は言います：

The default character set, which must be assumed in the absence of a charset parameter, is US-ASCII.

charsetパラメータが存在しない場合に想定されなければならないデフォルトの文字セットは、US-ASCIIです。

As explained in the Introduction section, this rule is considered outdated, so this document replaces it with the following set of rules:

はじめにセクションで説明したように、このルールは時代遅れと見なされるので、このドキュメントでは、次のルールセットでそれを置き換えます。

Each subtype of the "text" media type that uses the "charset" parameter can define its own default value for the "charset" parameter, including the absence of any default.

任意のデフォルトの不在を含め、「文字セット」パラメータに独自のデフォルト値を定義することができ、「文字セット」パラメータを使用して「テキスト」メディアタイプの各サブタイプ。 

   In order to improve interoperability with deployed agents, "text/*"
   media type registrations SHOULD either

a. specify that the "charset" parameter is not used for the defined subtype, because the charset information is transported inside the payload (such as in "text/xml"), or

A。文字セット情報（例えば、「テキスト/ XML」のような）ペイロードの内部に搬送されるので、「文字セット」パラメータは、定義されたサブタイプに使用されていないことを指定するか、または

b. require explicit unconditional inclusion of the "charset" parameter, eliminating the need for a default value.

B。デフォルト値の必要性を排除し、「文字セット」のパラメータを明示的に無条件含めることが必要です。 

   In accordance with option (a) above, registrations for "text/*" media
   types that can transport charset information inside the corresponding
   payloads (such as "text/html" and "text/xml") SHOULD NOT specify the
   use of a "charset" parameter, nor any default value, in order to
   avoid conflicting interpretations should the "charset" parameter
   value and the value specified in the payload disagree.

Thus, new subtypes of the "text" media type SHOULD NOT define a default "charset" value. If there is a strong reason to do so despite this advice, they SHOULD use the "UTF-8" [RFC3629] charset as the default.

このように、「テキスト」メディアタイプの新しいサブタイプは、デフォルトの「文字セット」の値を定義するべきではありません。このアドバイスにもかかわらず、そうする強い理由がある場合は、デフォルトとして「UTF-8」[RFC3629]の文字セットを使用すべきです。 

   Regardless of what approach is chosen, all new "text/*" registrations
   MUST clearly specify how the charset is determined; relying on the
   default defined in Section 4.1.2 of [RFC2046] is no longer permitted.
   However, existing "text/*" registrations that fail to specify how the
   charset is determined still default to US-ASCII.
        
   Specifications covering the "charset" parameter, and what default
   value, if any, is used, are subtype-specific, NOT protocol-specific.
   Protocols that use MIME, therefore, MUST NOT override default charset values for "text/*" media types to be different for their specific
   protocol.  The protocol definitions MUST leave that to the subtype
   definitions.
4. Default "charset" Parameter Value for "text/plain" Media Type
4.デフォルトの「文字セット」「text / plainの」メディアタイプのパラメータ値

The default "charset" parameter value for "text/plain" is unchanged from [RFC2046] and remains as "US-ASCII".

「プレーンテキスト/」デフォルト「のcharset」パラメータ値は[RFC2046]から変更されず、「US-ASCII」のままです。

5. Security Considerations
5.セキュリティについての考慮事項

Guessing of the "charset" parameter can lead to security issues such as content buffer overflows, denial of services, or bypass of filtering mechanisms. However, this document does not promote guessing, but encourages use of charset information that is specified by the sender.

「文字セット」パラメータを推測することは、そのようなコンテンツバッファオーバーフロー、サービス拒否、またはフィルタリングメカニズムのバイパスなどのセキュリティ上の問題につながることができます。しかし、この文書は推測促進が、送信者によって指定された文字セット情報の利用を奨励していません。

Conflicting information in-band vs. out-of-band can also lead to similar security problems, and this document recommends the use of charset information that is more likely to be correct (for example, in-band over out-of-band).

アウトオブバンド矛盾する情報インバンド対も同様のセキュリティ上の問題につながることができ、そしてこの文書は（例えば、インバンドオーバーアウト・オブ・バンド）が正しい可能性が高いです文字セット情報を使用することを推奨しています。
