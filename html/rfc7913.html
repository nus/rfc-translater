                   P-Access-Network-Info ABNF Update

Abstract

抽象

This document updates RFC 7315, by modifying the extension-access-info part of the P-Access-Network-Info header field Augmented Backus-Naur Form (ABNF), and by adding the following 'access-info' header field parameter values to the list of 'access-info' header field parameter values in the ABNF: 'operator-specific-GI' and 'utran-sai-3gpp'. The values are defined in the ABNF but are not included in the list.

このドキュメント更新RFC 7315、P-Access-Network-Infoヘッダフィールドた拡張バッカス・ナウアフォーム（ABNF）の拡張アクセス-info部を変更することにより、およびに次の「アクセス情報」ヘッダフィールドパラメータ値を加算し「演算子固有-GI」と「UTRAN-SAI-3GPP」：ABNFの「アクセス情報」ヘッダフィールドのパラメータ値のリスト。値はABNFで定義されていますが、リストには含まれません。

1. Introduction
1.はじめに

[RFC7315] defines the Session Initiation Protocol (SIP) [RFC3261] P-Access-Network-Info header field.

[RFC7315]は、セッション開始プロトコル（SIP）[RFC3261] P-Access-Network-Infoヘッダフィールドを定義します。

The header field syntax contains an 'access-info' header field parameter, with a set of pre-defined values. Most pre-defined values follow the 'generic-param' rule [RFC3261]. However, currently the Augmented Backus-Naur Form (ABNF) [RFC5234] in [RFC7315] requires new values to follow the 'gen-value' rule [RFC3261], which is more restrictive than 'generic-param'. There is no justification for the restriction, and new values have already been defined following the 'generic-param' rule.

ヘッダフィールドの構文は、事前定義された値のセットを使用して、「アクセス情報」ヘッダフィールドパラメータを含んでいます。ほとんどの事前定義された値は、「ジェネリック-のparam」ルール[RFC3261]を実行します。しかし、現在増補バッカス記法（ABNF）[RFC7315]の[RFC5234]は「一般的な-のparam」よりも限定的である「GEN-価値」ルール[RFC3261]を、従って新しい値を必要とします。そこ制限のための正当化はなく、新しい値がすでに「ジェネリック-のparam」ルール以下に定義されています。

The ABNF also defines the following 'access-info' header field parameter values: 'operator-specific-GI' and 'utran-sai-3gpp'. However, the values are not included in the list of 'access-info' values.

「オペレータ固有-GI」と「UTRAN-SAI-3GPP」：ABNFは、以下の「アクセス情報」ヘッダフィールドパラメータ値を定義します。しかし、値が「アクセス情報」の値のリストに含まれていません。

This document updates Section 5.4 of [RFC7315], by modifying the extension-access-info part of the P-Access-Network-Info header field ABNF.

この文書では、P-Access-Network-InfoヘッダフィールドのABNFの拡張アクセス-info部を修正することにより、[RFC7315]の5.4節を更新します。

As the P-Access-Network-Info header field is mainly used in networks defined by the 3rd-Generation Partnership Project (3GPP), where new values following the 'generic-param' rule have been defined [TS.3GPP.24.229], the update is not considered to cause issues with backward compatibility.

P-Access-Network-Infoヘッダフィールドは、主に「GENERIC-PARAM」ルール次の新しい値が[TS.3GPP.24.229]で定義されている第3世代パートナーシッププロジェクト（3GPP）によって定義されたネットワークで使用されるように、更新は、下位互換性の問題を引き起こすと考えられていません。

2. Update to
2. Updateへ

This section updates the ABNF defined in Section 5.4 of RFC 7315, as described below:

以下に説明するように、このセクションでは、RFC 7315のセクション5.4で定義されたABNFを更新します。

Old syntax:

古い構文： 

      access-info            = cgi-3gpp / utran-cell-id-3gpp /
                               dsl-location / i-wlan-node-id /
                               ci-3gpp2 / eth-location /
                               ci-3gpp2-femto / fiber-location /
                               np / gstn-location /local-time-zone /
                               dvb-rcs2-node-id / extension-access-info
      np                     = "network-provided"
      extension-access-info  = gen-value

New syntax:

新しい構文： 

      access-info            = cgi-3gpp / utran-cell-id-3gpp /
                               dsl-location / i-wlan-node-id /
                               ci-3gpp2 / eth-location /
                               ci-3gpp2-femto / fiber-location /
                               np / gstn-location /local-time-zone /
                               dvb-rcs2-node-id / operator-specific-GI /
                               utran-sai-3gpp / extension-access-info
      np                     = "network-provided"
      extension-access-info  = generic-param
3. Security Considerations
3.セキュリティの考慮事項

The security considerations for the P-Access-Network-Info header field are defined in [RFC7315]. The ABNF update defined in this document does not impact the security considerations.

P-Access-Network-Infoヘッダフィールドのセキュリティ問題は[RFC7315]で定義されています。この文書で定義されたABNFの更新はセキュリティ上の考慮事項に影響を与えません。
