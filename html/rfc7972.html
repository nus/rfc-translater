Abstract

抽象

Entertainment Identifier Registry (EIDR) Identifiers are used for the globally unique identification of motion picture and television content. This document defines the formal Uniform Resource Name (URN) Namespace Identifier (NID) for EIDR Identifiers.

エンターテインメント識別子レジストリ（EIDR）識別子は、映画やテレビコンテンツのグローバルにユニークな識別のために使用されます。この文書では、EIDR識別子のための正式なURN（Uniform Resource Name）の名前空間識別子（NID）を定義します。

1. Introduction
1.はじめに

This document defines the formal Uniform Resource Name (URN) Namespace Identifier (NID) for EIDR Identifiers.

この文書では、EIDR識別子のための正式なURN（Uniform Resource Name）の名前空間識別子（NID）を定義します。

EIDR Identifiers are centrally registered, globally unique identifiers for audiovisual works and their variants, primarily film and television. A unique set of metadata parameters describing the associated work is bound to the EIDR Identifier at the time of registration.

EIDR識別子は、中央視聴覚作品とその変種、主に映画やテレビのためのグローバル一意識別子、登録されています。関連する作業を記述するメタデータパラメータの固有のセットは、登録の時点でEIDR識別子にバインドされています。

EIDR Identifiers are a special kind of DOI Name allocated by EIDR Association specifically for audiovisual works. The Digital Object Identifier (DOI) system provides an infrastructure for persistent unique identification of objects of any type and is specified in [ISO26234]. A DOI Name is the identifier that is permanently assigned to an object. The use of DOI Names ensures persistence, global uniqueness, and an open resolution infrastructure.

EIDR識別子は、視聴覚作品のために特別にEIDR会によって割り当てられたDOI名の特別な種類です。デジタルオブジェクト識別子（DOI）システムは、任意のタイプのオブジェクトの持続的な固有の識別のためのインフラストラクチャを提供し、[ISO26234]で指定されています。 DOI名永久オブジェクトに割り当てられた識別子です。 DOI名の使用は、永続性、グローバルな一意性、およびオープン解像度のインフラストラクチャを実現します。

As a DOI Name, an EIDR Identifier has two components: a prefix assigned exclusively to EIDR Association by the DOI Registration Authority and a suffix defined by EIDR Association. Currently, EIDR Identifiers use the prefix "10.5240", but additional prefixes might be introduced in the future.

DOI登録機関によってEIDR協会に排他的に割り当てられたプレフィックスとEIDR協会によって定義されたサフィックス：DOI名として、EIDR識別子は、2つのコンポーネントがあります。現在、EIDR識別子は「10.5240」の接頭辞を使用しますが、追加のプレフィックスは、将来的に導入される可能性があります。

The following is an example of an EIDR Identifier in its canonical representation:

以下は、その正規表現でEIDR識別子の例です。

10.5240/7791-8534-2C23-9030-8610-5
10.5240 / 7791-8534-2sea-9030-8610-5

where "10.5240" is the prefix and "7791-8534-2C23-9030-8610-5" is the suffix.

ここで、「10.5240」は接頭辞で、「7791-8534-2C23-9030-8610-5は」接尾辞です。

The canonical URN representation of the same EIDR Identifier is:

同じEIDR識別子の正規URN表現です。

urn:eidr:10.5240:7791-8534-2C23-9030-8610-5

URN：EIDR：10.5240：7791-8534-2C23-9030-8610-5

Note that ":" is used as separator between the prefix and the suffix since "/" is a URN reserved character.

「/」はURN予約文字であるため、接頭辞と接尾辞の間にセパレータとして使用されている：「」ことに注意してください。

[EIDR-OVERVIEW] and [EIDR-INTRO] provide additional background information.

[EIDR-概要]と[EIDR-INTRO]追加の背景情報を提供します。

This document obsoletes RFC 7302. See Appendix A for a summary of changes.

この文書は、変更の概要については、RFC 7302を参照してください、付録Aを廃止します。

Declaration of syntactic structure:

構文構造の宣言：

An EIDR Identifier is a special kind of DOI Name (see [ISO26234]) and, as such, consists of two components: a prefix denoted EIDR-PREFIX and a suffix denoted EIDR-SUFFIX. The URN representation URN-EIDR of an EIDR Identifier conforms to the syntax (expressed using [RFC5234]):

EIDR識別子はDOI名の特別な種類である（[ISO26234]を参照）と、のような、2つのコンポーネントで構成さ：プレフィックスがEIDR-PREFIXを示され、サフィックスはEIDR-接尾辞を付し。 URN表現URN-EIDR EIDR識別子は、（[RFC5234]を使用して表される）構文に準拠しています。 

                   URN-EIDR = "urn:eidr:" EIDR-NSS
                   EIDR-NSS = EIDR-PREFIX ":" EIDR-SUFFIX
                   EIDR-PREFIX = 1*EIDR-CHARS
                   EIDR-SUFFIX = 1*EIDR-CHARS
                   EIDR-CHARS = ALPHA / DIGIT / "-" / "." / "_"

The only value of EIDR-PREFIX currently in use is "10.5240". Additional prefix values conforming to the EIDR-PREFIX syntax might be introduced by EIDR Association in the future. In all cases, each of these additional prefix values will be associated with suffix values that conform to a subset of the EIDR-SUFFIX syntax. These additional prefixes and their corresponding suffixes will be documented in future revisions to this registration.

現在使用されているEIDR-PREFIXの値のみが「10.5240」です。 EIDR-PREFIXの構文に準拠した追加のプレフィックス値は、将来的にEIDR協会によって導入される可能性があります。すべての場合において、これらの追加プレフィックス値のそれぞれは、EIDR-SUFFIX構文のサブセットに準拠サフィックス値に関連付けられます。これらの追加の接頭辞とそれに対応する接尾辞は、この登録の将来の改正に記載されます。

An implementation can process an EIDR-NSS with an unknown prefix as an opaque string per the "Rules of Lexical Equivalence" below, and resolve it as a generic DOI Name per the "Process for identifier resolution" below.

実装は、以下の「字句等価の規則」あたり不透明な文字列として、未知のプレフィックスでEIDR-NSSを処理し、以下の「識別子の解決のためのプロセス」あたりの一般的なDOI名としてそれを解決することができます。

When EIDR-PREFIX is equal to "10.5240", the syntax of EIDR-SUFFIX is further constrained according to the RP2079-SUFFIX syntax specified in [SMPTERP2079]:

EIDR-PREFIXが「10.5240」に等しい場合、EIDRサフィックスの構文はさらに[SMPTERP2079]で指定RP2079-サフィックス構文に従って制約されます。 

                   RP2079-SUFFIX = 5(4HEXDIG "-") CHECK
                   CHECK         = DIGIT / ALPHA

where CHECK is the Mod 37,36 check character as specified in [ISO7064], computed over the 20 hexadecimal digits HEXDIG of RP2079-SUFFIX.

20桁の16進数RP2079サフィックスのHEXDIGにわたって計算、[ISO7064]で指定されるようにCHECKはモジュレーション37,36チェックキャラクタです。

EXAMPLE: urn:eidr:10.5240:7791-8534-2C23-9030-8610-5

例：URN：EIDR：10.5240：7791-8534-2C23-9030-8610-5

Relevant ancillary documentation:

関連の補助ドキュメント：

[ISO26234] specifies DOI Name syntax and registration.

[ISO26234]はDOI名の構文や登録を指定します。

[SMPTERP2079] specifies the syntax of the EIDR Identifier with the prefix "10.5240".

[SMPTERP2079]接頭辞「10.5240」でEIDR識別子の構文を指定します。

Identifier uniqueness considerations:

識別子一意性の考慮事項：

An EIDR Identifier is a DOI Name. The ISO 26324 Registration Authority ensures DOI Name uniqueness; therefore, the URN-EIDR derived from an EIDR Identifier is guaranteed to be unique and never reassigned.

EIDR識別子は、DOIの名前です。 ISO 26324登録機関はDOI名の一意性を保証します。そのため、EIDR識別子由来URN-EIDRは一意であることが保証、決して再割り当てされます。

Furthermore, an EIDR Identifier is associated with a single URN-EIDR.

更に、EIDR識別子が単一URN-EIDRと関連しています。

Identifier persistence considerations:

識別子の永続性の考慮事項：

An EIDR Identifier is a DOI Name. The ISO 26324 Registration Authority ensures that a DOI Name remains valid indefinitely; therefore, the URN-EIDR derived from an EIDR Identifier remains valid indefinitely.

EIDR識別子は、DOIの名前です。 ISO 26324登録機関はDOI名は無期限に有効なままであることを保証します。従って、EIDR識別子に由来URN-EIDRは無期限に有効です。

Process of identifier assignment:

識別子割り当てのプロセス。

EIDR Association registers each EIDR Identifier with the ISO 26324 Registration Authority.

EIDR協会は、ISO 26324登録機関と各EIDR識別子を登録します。

Entities involved in the motion picture and television industry, either directly in the production of content or in its distribution and related services, are eligible to apply for registrant status. Registrants pay an annual membership fee that depends on the size of the company.

映画やテレビ業界では、直接コンテンツの制作やその分布との関連サービスに関与するエンティティは、登録者のステータスを申請する資格があります。登録者は、会社の規模に依存年会費を支払います。

Process for identifier resolution:

識別子の解決のためのプロセス。

As a DOI Name, the resolution of the EIDR Identifier associated with an EIDR-NSS is handled by the ISO 26324 Registration Authority.

DOI名として、EIDR-NSSに関連付けられたEIDR識別子の解像度は、ISO 26324登録機関によって処理されます。

The ISO 26324 Registration Authority operates a web service that allows the EIDR Identifier associated with an EIDR-NSS to be resolved by issuing an HTTP GET request to the following URI (see [RFC7231] and [RFC2818]):

ISO 26324登録機関は、次のURIに対してHTTP GETリクエストを発行することによって解決されるEIDR-NSSに関連付けられたEIDR識別子（[RFC7231]及び[RFC2818]を参照）ことができ、ウェブサービスを運営します。

"https://doi.org/" URN-EIDR

"Https://doi.org/" URN EIDR

If the EIDR Identifier exists, then an XML Schema instance document (see [XMLSchema]) containing metadata associated with the EIDR Identifier is returned; otherwise, the HTTP status code "404 Not Found" is returned.

EIDR識別子が返されるに関連付けられたメタデータを含むEIDR識別子が存在する場合、XMLスキーマのインスタンス文書（[XMLスキーマ]を参照）。それ以外の場合は、HTTPステータスコードが「404が見つかりません」が返されます。

EXAMPLE: https://doi.org/urn:eidr:10.5240:7791-8534-2C23-9030-8610-5

例：https://doi.org/urn:eidr:10.5240:7791-8534-2C23-9030-8610-5

NOTE: The web service uses the HTTP Accept header to determine the format of the response. When using a web browser to access the resource above, the query string "?locatt=type:Full" is appended to the URI to explicitly request an XML response.

注：Webサービスは、HTTP応答の形式を決定するために、ヘッダーを受け入れ使用しています。上記のリソースにアクセスするには、Webブラウザを使用する場合は、クエリ文字列「locatt =タイプ：？フル」を明示的XMLレスポンスを要求するためにURIに追加されます。

EIDR Association operates additional services specifically tailored to EIDR users (see [EIDR-SERVICES] and [DOI-EIDR-PROXY]). These services include additional features, e.g., the ability to register EIDR Identifiers and web-based GUI tools.

EIDR協会は、具体的（参照[EIDR-SERVICES]と[DOI-EIDR-PROXY]）ユーザをEIDRに合わせた追加サービスを運営しています。これらのサービスには追加機能が含まれ、例えば、EIDR識別子およびWebベースのGUIツールを登録する機能。

Rules for Lexical Equivalence:

字句等価のルール：

Lexical equivalence of the URN-EIDR is defined by case-insensitive string match.

URN-EIDRの語彙等価は、大文字と小文字を区別しない文字列の一致によって定義されます。

NOTE: The URN-EIDR includes the "urn:eidr:" string, as specified under "Declaration of syntactic structure".

注：URN-EIDRが含まれて「：EIDR：URNを」文字列「構文構造の宣言」の下に指定されています。

Conformance with URN Syntax:

URN構文に準拠：

As specified above, the syntax of the URN-EIDR is a subset of the URN syntax specified in [RFC2141].

上記指定されたように、URN-EIDRの構文は、[RFC2141]で指定されたURN構文のサブセットです。

Validation mechanism:

検証メカニズム：

The validity of an EIDR-NSS can only be guaranteed by completing the resolution process.

EIDR-NSSの有効性は、唯一の解決プロセスを完了することによって保証することができます。

For EIDR-NSS with an EIDR-PREFIX equal to "10.5240", the CHECK value can be used for integrity checking, as specified above.

上記で指定される「10.5240」に等しいEIDR-prefixでEIDR-NSSのために、チェック値は、完全性検査のために使用することができます。

Scope:

範囲：

EIDR Identifiers are centrally registered, globally unique identifiers for use with audiovisual works worldwide.

EIDR識別子は中心部、世界的な視聴覚作品で使用するためのグローバル一意識別子を登録されています。

3. Namespace Considerations
3.名前空間の考慮事項

EIDR Identifiers are intended for use in Internet applications, where URNs are routinely used to identify audiovisual resources. There is no direct mapping from EIDR Identifiers to existing URN namespaces.

EIDR識別子はURNのが日常視聴覚リソースを識別するために使用されているインターネットアプリケーションでの使用を目的としています。既存のURN名前空間へのEIDR識別子からの直接マッピングはありません。

4. Community Considerations
4.コミュニティの考慮事項

The primary registrants of EIDR Identifiers are producers and distributors of audiovisual works, metadata aggregators, and audiovisual archives. Any bona fide member of the ecosystem can become an EIDR member and register any work. EIDR Identifiers can be used by anyone to unambiguously identify an audiovisual asset and retrieve underlying metadata. The primary benefits of their use are associated with works in wide distribution, management of long or complex distribution chains, and aggregation of information from multiple sources.

EIDR識別子の主な登録者は、生産者や流通業者視聴覚作品の、メタデータアグリゲータ、及び視聴覚アーカイブです。生態系のいずれかの善意のメンバーは、EIDRメンバーになると、すべての作業を登録することができます。 EIDR識別子は明確視聴覚資産を特定し、根本的なメタデータを取得するために誰でも使用することができます。これらの使用の主な利点は、広い分布の作品、長いまたは複雑な流通チェーンの管理、および複数のソースからの情報の集約と関連しています。

6. Security Considerations
6.セキュリティの考慮事項

This document specifies the syntax of the URN-EIDR namespace and makes no security representations. Furthermore, the details of assignment and resolution processes are defined external to this document by EIDR Association and the ISO 26324 Registration Authority, and are thus outside the scope of this document. Note, however, that failure to conform to the syntactic and lexical equivalence rules in this specification when using an EIDR Identifier as a criteria for accessing restricted resources can result in granting unauthorized access to these resources.

この文書では、URN-EIDR名前空間の構文を指定し、一切のセキュリティ表明を行いません。また、割当及び解像度処理の詳細は、EIDR協会及びISO 26324登録機関により本文書の外部に定義され、この文書の範囲外従っています。ただし、制限されたリソースにアクセスするための基準としてEIDR識別子を使用している場合、この仕様では、構文や語彙の等価性規則に準拠するために、その失敗は、これらのリソースへの不正アクセスを許可するにつながることができます。
