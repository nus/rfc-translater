            IANA Considerations for IPv6 Neighbor Discovery
                    Prefix Information Option Flags

Abstract

抽象

The Prefix Information Option (PIO) in the IPv6 Neighbor Discovery Router Advertisement message defines an 8-bit flag field; this field has two flags defined, and the remaining 6 bits are reserved (Reserved1). RFC 6275 defines a flag from this field without creating an IANA registry or updating RFC 4861. The purpose of this document is to create an IANA registry for the PIO flags. This document updates RFC 4861.

IPv6近隣探索ルータ広告メッセージ内のプレフィックス情報オプション（PIO）は、8ビットのフラグフィールドを定義します。このフィールドは定義された2つのフラグを有し、残りの6ビット（Reserved1）が予約されています。 RFC 6275は、この文書の目的は、PIOフラグ用のIANAレジストリを作成することであるRFC 4861.をIANAレジストリを作成または更新することなく、このフィールドのフラグを定義します。この文書は、RFC 4861に更新します。

1. Introduction
1.はじめに

The Prefix Information Option (PIO) in the IPv6 Neighbor Discovery Router Advertisement message defines an 8-bit flag field; this field has two flags defined and the remaining 6 bits are reserved (Reserved1). RFC 6275 defines a flag from this field without creating an IANA registry or updating RFC 4861. The purpose of this document is to create a new registry for the PIO flags.

IPv6近隣探索ルータ広告メッセージ内のプレフィックス情報オプション（PIO）は、8ビットのフラグフィールドを定義します。このフィールドは2つの定義されたフラグと、残りの6ビットは予約されている（Reserved1）を有しています。 RFC 6275は、この文書の目的は、PIOフラグのための新しいレジストリを作成することであるRFC 4861.をIANAレジストリを作成または更新することなく、このフィールドからフラグを定義します。

2. Current Prefix Information Option Flags
2.現在のプレフィックス情報オプションフラグ

Currently, the Neighbor Discovery Protocol Prefix Information Option [RFC4861] contains the following one-bit flags defined in published RFCs:

現在、近隣探索プロトコルのプレフィックス情報オプション[RFC4861]は、公開RFCで定義された次の1ビットのフラグが含まれています。 

                               0 1 2 3 4 5 6 7
                              +-+-+-+-+-+-+-+-+
                              |L|A|R|Reserved1|
                              +-+-+-+-+-+-+-+-+

Figure 1: PIO Flags

図1：PIOフラグ

L - On-link Flag [RFC4861]

L - オンリンクフラグ[RFC4861]

A - Autonomous Address Configuration Flag [RFC4861]

- 自律アドレス設定フラグ[RFC4861]

R - Router Address Flag [RFC6275]

R - ルータアドレスフラグ[RFC6275]

Reserved1 - Reserved

Reserved1 - 予約

3. Updates to
3.アップデートへ

This document updates Section 4.6.2 "Prefix Information" of [RFC4861] to point to the IANA registry that is created in this document (see Section 4).

この文書は（セクション4を参照）、この文書で作成されたIANAレジストリを指すように、[RFC4861]のセクション4.6.2「プレフィックス情報」を更新します。

Specifically, the current list of flags in the Prefix Information Option can be found in the "IPv6 Neighbor Discovery Prefix Information Option Flags" registry.

具体的には、プレフィックス情報オプションのフラグの現在のリストは、「IPv6の近隣探索プレフィックス情報オプションフラグ」のレジストリで見つけることができます。

4. IANA Considerations
4. IANAの考慮事項

IANA has created a new registry for IPv6 Neighbor Discovery Prefix Information Option flags. This registry includes the current flags in the PIO. The initial contents of the registry are as follows:

IANAは、IPv6近隣探索プレフィックス情報オプションフラグのための新しいレジストリを作成しました。このレジストリは、PIOの現在のフラグを含んでいます。次のようにレジストリの初期の内容は以下のとおりです。 

       +----------------+---------------------------------+-----------+
       | PIO Option Bit | Description                     | Reference |
       +----------------+---------------------------------+-----------+
       | 0              | L - On-link Flag                | [RFC4861] |
       | 1              | A - Autonomous Address          | [RFC4861] |
       |                |     Configuration Flag          |           |
       | 2              | R - Router Address Flag         | [RFC6275] |
       | 3-7            | Reserved                        |           |
       +----------------+---------------------------------+-----------+

Figure 2: IPv6 Neighbor Discovery Prefix Information Option Flags

図2：IPv6の近隣探索プレフィックス情報オプションフラグ

The assignment of new flags in the PIO option header requires Standards Action [RFC8126].

PIOオプションヘッダの新しいフラグの割り当ては標準アクション[RFC8126]を必要とします。

The registry for these flags is available at <http://www.iana.org/assignments/icmpv6-parameters>.

これらのフラグのレジストリは、<http://www.iana.org/assignments/icmpv6-parameters>でご利用いただけます。
