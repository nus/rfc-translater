        Registration of Second-Level URN Namespaces under "ietf"

Abstract

抽象

RFC 2648 defines the "ietf" URN namespace and a number of sub-namespaces. RFC 3553 defines an additional sub-namespace, "params", and creates a registry to document allocations under that. But there is no registry that lists, in one place, all sub-namespaces of "ietf". This document creates and populates such a registry, thereby changing the mechanism defined in RFC 2648 for adding new sub-namespaces of "ietf".

RFC 2648は、「IETF」URN名前空間と副名前空間の数を定義します。 RFC 3553は、追加のサブ名前空間、「paramsは」を定義し、その下の割り当てを文書化するために、レジストリを作成します。しかし、一つの場所で、一覧表示されます何のレジストリ、「IETF」のすべてのサブ名前空間がありません。この文書は、それによって「IETF」の新たなサブ名前空間を追加するためのRFC 2648で定義された機構を変更する、そのようなレジストリを作成し、移入します。

1. Introduction
1. はじめに

"A URN Namespace for IETF Documents" [RFC2648] defines the "ietf" URN namespace and a number of sub-namespaces. "An IETF URN Sub-namespace for Registered Protocol Parameters" [RFC3553] defines an additional sub-namespace, "params", and creates a registry to document allocations under that. But there is no registry that lists, in one place, all sub-namespaces of "ietf". This document creates and populates such a registry, thereby changing the mechanism defined in RFC 2648 for adding new sub-namespaces of "ietf".

「IETF文書のURN名前空間」[RFC2648]は、「IETF」URN名前空間と副名前空間の数を定義します。 「IETF URN登録プロトコル・パラメータのためのサブ名前空間」[RFC3553]は追加のサブ名前空間、「paramsは」を定義し、その下の割り当てを文書化するために、レジストリを作成します。しかし、一つの場所で、一覧表示されます何のレジストリ、「IETF」のすべてのサブ名前空間がありません。この文書は、それによって「IETF」の新たなサブ名前空間を追加するためのRFC 2648で定義された機構を変更する、そのようなレジストリを作成し、移入します。

2. IANA Considerations
2. IANAの考慮事項

There is currently a top-level registry group called "IETF Protocol Parameter Identifiers", which contains one registry, "IETF URN Sub-namespace for Registered Protocol Parameters". IANA has taken the following three actions:

1つのレジストリ、「登録プロトコル・パラメータのためのIETF URNサブ名前空間」を含む「IETFプロトコルパラメータ識別子」と呼ばれるトップレベルのレジストリ・グループは、現在あり。 IANAは、次の3つのアクションをとっています。

Action 1: Renamed the group "IETF Protocol Parameter Identifiers", giving it the new name "Uniform Resource Name (URN) Namespace for IETF Use". The existing registry in that group remains, and its name is unchanged. Its registration procedure has been updated to "IETF Review" (formerly called "IETF Consensus").

アクション1：それに新しい名前「IETFの使用のための統一リソース名（URN）名前空間」を与えるグループ「IETFプロトコルパラメータ識別子」を、改名。そのグループ内の既存のレジストリが残っており、その名前は変更されません。その登録手順は、「IETFレビュー」（旧称「IETFコンセンサス」）に更新されました。

Action 2: Added a new registry to the renamed group. The new registry is called "IETF URN Sub-namespaces", and new registrations will use the IETF Review policy [RFC5226], which provides for IETF consensus in order to add a new URN namespace under "ietf". The new registry appears first in the group, to provide a human-friendly, top-down resolution of the namespace hierarchy.

アクション2：名前を変更したグループに新しいレジストリを追加しました。新しいレジストリは、「IETF URNサブ名前空間」と呼ばれ、新たな登録は、「IETF」の下に新しいURN名前空間を追加するために、IETFコンセンサスを提供IETFレビューポリシー[RFC5226]を使用します。新しいレジストリは、名前空間階層の人間に優しい、トップダウンの分解能を提供するために、最初のグループに表示されます。

Action 3: Populated the new registry as follows:

アクション3：次のように新しいレジストリを移入：

IETF URN Sub-namespaces

IETF URNサブ名前空間

Registration Procedures: IETF Review Reference: [RFC6924]

登録手順：IETFレビュー参考：[RFC6924]

Note: This is the Official Registry for sub-namespaces of the 'IETF' URN Namespace.

注：これは、「IETFのURN名前空間のサブ名前空間の公式レジストリです。

   mtg           | [RFC2648] | none
