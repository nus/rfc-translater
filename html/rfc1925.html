                      The Twelve Networking Truths

Status of this Memo

このメモの位置づけ

This memo provides information for the Internet community. This memo does not specify an Internet standard of any kind. Distribution of this memo is unlimited.

このメモは、インターネットコミュニティに情報を提供します。 このメモは、いかなる種類のインターネット標準も指定していません。 このメモの配布は無制限です。

Abstract

概要

This memo documents the fundamental truths of networking for the Internet community. This memo does not specify a standard, except in the sense that all standards must implicitly follow the fundamental truths.

このメモは、インターネットコミュニティのネットワーキングの基本的な真実を文書化したものです。 このメモは、すべての標準が暗黙のうちに基本的な真理に従わなければならないという意味を除いて、標準を指定しません。

1. Introduction
1. はじめに

This Request for Comments (RFC) provides information about the fundamental truths underlying all networking. These truths apply to networking in general, and are not limited to TCP/IP, the Internet, or any other subset of the networking community.

このRequest for Comments（RFC）は、すべてのネットワーキングの根底にある基本的な真実に関する情報を提供します。 これらの真実は、一般のネットワークに適用され、TCP/IP、インターネット、またはネットワーキングコミュニティの他のコミュニティに限定されません。

2. The Fundamental Truths
2. 基本真理

(1) It Has To Work.

(1) 動かなければならない。

(2) No matter how hard you push and no matter what the priority, you can't increase the speed of light.

(2) どれだけ強く押しても、どんな優先事項があろうとも、光の速さを超えることはできない。

(2a) (corollary). No matter how hard you try, you can't make a baby in much less than 9 months. Trying to speed this up *might* make it slower, but it won't make it happen any quicker.

(2a) (必然的結果) どんなに一生懸命努力しても、9か月未満で赤ちゃんを作ることはできない。 これを高速化しようとすると、遅くなることがあっても、速くなることはない。

(3) With sufficient thrust, pigs fly just fine. However, this is not necessarily a good idea. It is hard to be sure where they are going to land, and it could be dangerous sitting under them as they fly overhead.

(3) 十分な推力があれば、豚はうまく飛ぶ。 ただし、これは必ずしも良い考えではない。 彼らがどこに着陸するのかを確認するのは困難であり、彼らが頭上を飛ぶとき、彼らの下に座っているのは危険である。

(4) Some things in life can never be fully appreciated nor understood unless experienced firsthand. Some things in networking can never be fully understood by someone who neither builds commercial networking equipment nor runs an operational network.

(4) 直接経験するまでは、完全に理解できないことが人生にはある。 商用のネットワーク機器を構築したり、運用ネットワークを運用したことがない人は、ネットワークの一部を完全に理解することはできない。

(5) It is always possible to aglutenate multiple separate problems into a single complex interdependent solution. In most cases this is a bad idea.

(5) 複数の独立した問題を、相互依存する複雑な一つの解決法に集約することは常に可能であるが、ほとんどの場合、これは悪い考えである。

(6) It is easier to move a problem around (for example, by moving the problem to a different part of the overall network architecture) than it is to solve it.

(6) 問題を解決するよりも、問題を移動する方が簡単である（例えば、ネットワークアーキテクチャ全体とは別の場所に問題を移動すること）。

(6a) (corollary). It is always possible to add another level of indirection.

(6a) (必然的結果) 別のレベルの間接的手法を追加することは常に可能である。

(7) It is always something

(7) いつも何かある。

(7a) (corollary). Good, Fast, Cheap: Pick any two (you can't have all three).

(7a) (必然的結果) 良い、早い、安い: どれか2つを選んでください (3つ全てを選ぶことはできない)

(8) It is more complicated than you think.

(8) 想像以上に物事は複雑だ。

(9) For all resources, whatever it is, you need more.

(9) 全ての資源は、何であろうと、もっと必要になる。

(9a) (corollary) Every networking problem always takes longer to solve than it seems like it should.

(9a) (必然的結果) 全てのネットワークの問題は、解決するのに予想以上の時間がかかる。

(10) One size never fits all.

(10) 1つのサイズが全てに適合することはない。

(11) Every old idea will be proposed again with a different name and a different presentation, regardless of whether it works.

(11) すべての古いアイデアは、機能するかどうかに関係なく、異なる名前と異なる表現で再び提案される。

(11a) (corollary). See rule 6a.

(11a) (必然的結果) ルール 6a を参照。

(12) In protocol design, perfection has been reached not when there is nothing left to add, but when there is nothing left to take away.

(12) プロトコル設計においては、何も追加するものがないときではなく、何も取り除くものがないときに、理想のプロトコルになる。

Security Considerations

セキュリティの考慮事項

This RFC raises no security issues. However, security protocols are subject to the fundamental networking truths.

このRFCはセキュリティの問題を取り上げません。 ただし、セキュリティプロトコルは、基本的なネットワークの真実に左右されます。
